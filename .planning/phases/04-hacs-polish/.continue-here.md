<context>
The user asked about adding predictive sensors — specifically, a way to tell
homeowners when the cheapest time in the next 24 hours would be to run power-
hungry appliances like pool pumps or AC units. This requires:

1. IESO predispatch/forecast data (they publish day-ahead and pre-dispatch prices)
2. A forecast coordinator that fetches 24h of predicted prices
3. A "best_time_to_run" sensor that computes the cheapest multi-hour window
   (e.g. "2:00 AM - 6:00 AM" when prices are typically lowest)
4. Possibly a duration parameter in options (how long the appliance runs)

IESO data sources to investigate:
- PUB_PriceHOEPPredispOR.xml (pre-dispatch hourly prices)
- PUB_PredispatchHourlyOntarioZonalPrice.xml (post-MRP nodal predispatch)
- Day-ahead market results

The existing coordinator architecture is clean and supports adding a second
coordinator for forecast data easily.
</context>
